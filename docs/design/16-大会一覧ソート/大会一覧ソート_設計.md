## 目的

大会一覧([/tournaments/](https://ga-card-tools-jp.pages.dev/tournaments/))を**列見出しのクリックで昇順・降順に並べ替え**られるようにする(ユーザー改善要望・2026-07-22)。
現状は**開催日の降順で固定**([build-tournament-pages.mjs:376](../../../scripts/build-tournament-pages.mjs#L376))で並べ替え手段がない。

## ユーザー確定事項(2026-07-24)

| 論点 | 決定 |
|---|---|
| ソート対象の列 | **全6列**(開催日・大会名・開催国・フォーマット・種別・参加者数) |
| ソート状態のURL永続化 | **する**(既存の絞り込みと同じく共有URL・大会詳細からのブラウザバックで復元) |
| ソートUIの方式 | **PC=列見出しクリック(▲▼表示)/ スマホ=セレクトボックス**(スマホは `thead` 非表示のため見出しクリック不可) |

## 既存実装の要点(コード確定)

- 一覧テーブル `#ev-table`。各行 `<tr class="tr-link" data-...>` は [build-tournament-pages.mjs:548-561](../../../scripts/build-tournament-pages.mjs#L548) で生成。既存の `data-*`: `href` / `cat` / `country` / `format` / `season` / `date`(`YYYY-MM-DD`) / `search`
- 列: 開催日 `.date` / 大会名 `.ev-name`+主催 `.ev-host` / 開催国 `.country` / フォーマット `.format` / 種別 `.cat`(バッジ) / 参加 `.num`(`N名`)
- クライアントJSは `LIST_JS`([:1006-1106](../../../scripts/build-tournament-pages.mjs#L1006))として `/tournaments/tournaments.js` に出力
  - 絞り込みは `apply()`(行の `hidden` を切替)。**絞り込み状態は `saveQuery()` が `URLSearchParams`+`history.replaceState` でURLに書き戻し**、`loadQuery()` が復元([:1049-1078](../../../scripts/build-tournament-pages.mjs#L1049))
  - 大会詳細への各リンクにも同じクエリを付け直す([:1060-1065](../../../scripts/build-tournament-pages.mjs#L1060))。`DECK_JS` はパンくずの一覧リンクに `location.search` を引き継ぐ([:1119-1122](../../../scripts/build-tournament-pages.mjs#L1119))
- スマホ(≤640px)は `thead { display:none }`([:967](../../../scripts/build-tournament-pages.mjs#L967))でカード表示。**列見出しクリックが使えない**
- 参加者数 `playerCount` は常に数値([:233](../../../scripts/build-tournament-pages.mjs#L233))
- 種別の階層順は `CATEGORY_JP` の定義順([:36-40](../../../scripts/build-tournament-pages.mjs#L36)): `store-championships` → `regionals` → `nationals` → `ascent`

### 変更1: 行にソート用データ属性を付与(build時)

セルのテキスト(`N名`・種別バッジのネスト markup)を実行時パースするのは壊れやすいので、
**build時にソート値を明示的な `data-*` で持たせる**。行テンプレート([:550](../../../scripts/build-tournament-pages.mjs#L550))に追加:

| 列キー | 属性 | 値 | 比較 |
|---|---|---|---|
| `date` | `data-date`(**既存を流用**) | `YYYY-MM-DD` | 文字列(ISOなので辞書順=時系列) |
| `name` | `data-name`(新) | `esc(e.name)` | 文字列 `localeCompare('ja')` |
| `country` | `data-country-sort`(新) | 開催国の**プレーン表示名**(`countryLabelText(e.host.country)` [:629](../../../scripts/build-tournament-pages.mjs#L629)) | 文字列 |
| `format` | `data-format-sort`(新) | `formatJp(e.format)` | 文字列 |
| `cat` | `data-cat-sort`(新) | 階層ランク(下記) | 数値 |
| `players` | `data-players`(新) | `e.playerCount ?? 0` | 数値 |

- 種別ランクは `CATEGORY_JP` のキー順から生成する(例: `const CAT_RANK = Object.fromEntries(Object.keys(CATEGORY_JP).map((k, i) => [k, i]))` → store=0, regionals=1, nationals=2, ascent=3)。**Ascentが最上位**なので降順で上位大会が上に来る
- ⚠️ `countryLabelText` が `buildListHtml` から参照できるスコープか確認する。参照できなければ**同じプレーン名を返すヘルパを使う**(セル表示 `countryLabel` は旗などの markup を含み得るためソート値には使わない)
- 参加者数が不明な大会は `0` として最小(降順で末尾)に置く

### 変更2: 列見出しをクリック可能に(PC)

`thead`([:606](../../../scripts/build-tournament-pages.mjs#L606))を、`data-sort` キー付き・`aria-sort` 対応の**ボタン入り見出し**にする:

```html
<thead><tr>
  <th data-sort="date" aria-sort="descending"><button type="button" class="th-sort">開催日</button></th>
  <th data-sort="name"><button type="button" class="th-sort">大会名</button></th>
  <th data-sort="country"><button type="button" class="th-sort">開催国</button></th>
  <th data-sort="format"><button type="button" class="th-sort">フォーマット</button></th>
  <th data-sort="cat"><button type="button" class="th-sort">種別</button></th>
  <th data-sort="players"><button type="button" class="th-sort">参加</button></th>
</tr></thead>
```

- 初期は `date` に `aria-sort="descending"`(既定=開催日の新しい順)
- ▲▼インジケータは **CSSの `::after` + `aria-sort` 属性セレクタ**で出す(CSPの `style-src 'self'` に抵触しない。inline style は使わない):
  ```css
  th[data-sort] { cursor: pointer; }
  .th-sort { font: inherit; color: inherit; background: none; border: none; padding: 0; cursor: pointer; }
  th[data-sort] .th-sort::after { content: "⇅"; opacity: .35; margin-left: 4px; }
  th[aria-sort="ascending"]  .th-sort::after { content: "▲"; opacity: 1; }
  th[aria-sort="descending"] .th-sort::after { content: "▼"; opacity: 1; }
  ```

### 変更3: スマホ用のソート select

`thead` が隠れるスマホ向けに、**折りたたまれる `filter-box` の外**(件数 `#f-hits` の近く。畳んでも見える位置)にセレクトを置く。
`#f-hits`([:603](../../../scripts/build-tournament-pages.mjs#L603))の直後あたり:

```html
<div class="sort-mobile">
  <label for="f-sort">並べ替え</label>
  <select id="f-sort" aria-label="一覧の並べ替え">
    <optgroup label="開催日">
      <option value="date-desc">新しい順</option>
      <option value="date-asc">古い順</option>
    </optgroup>
    <optgroup label="参加者数">
      <option value="players-desc">多い順</option>
      <option value="players-asc">少ない順</option>
    </optgroup>
    <optgroup label="大会名"><option value="name-asc">A→Z</option><option value="name-desc">Z→A</option></optgroup>
    <optgroup label="開催国"><option value="country-asc">昇順</option><option value="country-desc">降順</option></optgroup>
    <optgroup label="フォーマット"><option value="format-asc">昇順</option><option value="format-desc">降順</option></optgroup>
    <optgroup label="種別"><option value="cat-desc">上位から</option><option value="cat-asc">下位から</option></optgroup>
  </select>
</div>
```

CSS: **既定は非表示、スマホでのみ表示**(PCは見出しクリックを使う):

```css
.sort-mobile { display: none; }
@media (max-width: 640px) {
  .sort-mobile { display: flex; align-items: center; gap: 8px; margin: 0 0 10px; }
}
```

### 変更4: LIST_JS にソートロジックを追加

- **列設定**(キー→属性・数値か):
  `date:{attr:"date",num:false}` / `name:{attr:"name",num:false}` / `country:{attr:"countrySort",num:false}` /
  `format:{attr:"formatSort",num:false}` / `cat:{attr:"catSort",num:true}` / `players:{attr:"players",num:true}`
  (`dataset` 名は属性名のキャメルケース。`data-country-sort` → `dataset.countrySort`)
- **既定方向**(初回クリック時): `date`/`players`/`cat` は `desc`、`name`/`country`/`format` は `asc`
- **安定化のためのタイブレーク**: `rows` を取得時のDOM順(=既定の開催日降順・id降順)で**元インデックスを記憶**し、キーが同値のときは元インデックスで比較する(現在のDOM順に依存せず決定的にする)
- **`sortBy(key, dir)`**:
  1. `rows` を比較関数で並べ替え、`tbody.append(...並べ替え後)` でDOMを再配置(`append` は既存要素を移動。`hidden` 状態は各要素に残るので絞り込みと独立)
  2. 全 `th[data-sort]` の `aria-sort` を更新(対象列だけ `ascending`/`descending`、他は除去)
  3. スマホ select の値を `key + "-" + dir` に同期(見出しクリックで変えても select が追従)
  4. `saveQuery()` を呼ぶ
- **比較関数**: `va,vb` を属性から取得。数値列は `Number` 比較、文字列列は `localeCompare(vb, 'ja')`。方向で符号反転。同値なら元インデックス昇順
- **イベント**:
  - `th[data-sort]` クリック/Enter → `key`。`key===sortKey` なら方向反転、違えばその列の既定方向で `sortBy`
  - `#f-sort` の `change` → 値 `"key-dir"` を分解して `sortBy`
- **URL永続化**(既存 `saveQuery`/`loadQuery` を拡張):
  - `saveQuery()`: `sortKey+"-"+sortDir` が既定(`date-desc`)**以外**のとき `p.set("sort", ...)`。既定なら書かない(URLを汚さない)
  - `loadQuery()`: `sort` を読み、`"key-dir"` が既知の組合せなら `sortKey/sortDir` に反映。**初期化順**は `loadQuery()` → `sortBy(sortKey, sortDir)`(並べ替え適用)→ `apply()`(絞り込み)
  - `popstate` でも同様に復元(既存の `addEventListener("popstate", ...)` に相乗り)

### 検証・再生成の手順

- 再生成は **`node scripts/build-tournament-pages.mjs --no-scan`**(APIを叩かず既存データからページ+`tournaments.js`を再生成)。
  ⚠️ CLAUDE.md記載: **`--no-scan` でも `data/tournaments/index.json` の `updatedAt` が毎回書き換わる**。
  **このタイムスタンプだけの差分はコミットに含めない**(`git checkout -- data/tournaments/index.json`)
- `npm run dev`(:3000)+ Playwright 実駆動(証明書回避 `--ignore-certificate-errors` + `ignoreHTTPSErrors:true`)
- 種別・参加者数・複数同日開催などの検証は既存の実データで再現できるはず(収録済み大会に多様性あり)

## 設計からの逸脱がないことの確認事項(実装担当向け)

- 新規 `data-*` は行あたり4個増える(`index.html` は約557KB)。**サイズ増は軽微**の想定だが、生成後に極端に膨らんでいないか確認
- `countryLabelText`(プレーン名)を使う。**旗などの markup を含む表示用ラベルをソート値にしない**
- 既存の絞り込み(`apply`/`saveQuery`/`loadQuery`)の**挙動を壊さない**。ソートは `fields`(入力要素の対応表)とは別系統で持ち、`saveQuery`/`loadQuery` に相乗りさせる
- 既存の「行全体クリックで大会へ遷移」([:1091-1096](../../../scripts/build-tournament-pages.mjs#L1091))は**見出しボタンのクリックと干渉しない**(見出しは `thead` 内、遷移ハンドラは `tbody` の行のみ)ことを確認
