## 決定済み要件（2026-07-16 ユーザー決定。モックレビュー2巡で確定）

- **表示データ**: ①エレメント別枚数 ②タイプ/サブタイプ内訳 ③Floating Memoryキーワード統計 ④フォーマット適合チェック
  - コストカーブは今回**見送り**（将来追加は容易。集計関数に1ブロック足すだけ）
- **表示場所**: 編集画面（#view-editor）と共有閲覧画面（#view-deck）の両方
- **UI形式**: タブ切替（「🃏 デッキ｜📊 統計」）。タブ基盤は新規作成
- **統計パネルの構成順（上から）**: マテリアルデッキ → メインデッキ → サイドボード → 合計 → フォーマット適合
- **合計ブロック** = マテリアル+メイン+サイドの全カード合算。「検討中」は除外
- **分類語の表記は「英語（日本語訳）」形式**（例: `WIND（風）`）。既存 `GA_I18N.label(kind, value)` の出力形式そのままでよい
- **⚠️警告バナー**: 禁止カードがあるデッキでは統計パネル最上部にバナー表示（フォーマット適合が最下部のため）。通常時は非表示
- **Floating Memoryの条件付き付与は内数表示（第5版）**: 例 `Floating Memory（フローティングメモリー） 9（うち条件付き 3枚）`。
  「条件付き」= [Class Bonus]/[Level N+]/[Vanitas Bonus]等の条件タグつきで付与されるもの

## 設計判断（実装前にユーザー確認済み）

1. **集計対象ゾーン**: エレメント別・タイプ/サブタイプ別・キーワードは material / main / side の3ゾーン各ブロック+合計ブロック。検討中(maybe)は常に除外
2. **編集画面のタブ挙動**: 検索バー(search-top)は統計タブ表示中も**出したまま**。統計を見ながらカード追加→集計が即時更新される流れを活かす（例:「ノームをあと2枚増やす」調整）
3. **フォーマット表示はスタンダード/パンテオンの2つ**。ドラフトは構築デッキに無意味なので出さない。判定対象はメイン+マテリアル+サイド（大会ではサイドも判定対象）
4. **複数エレメント/サブタイプ持ちのカードは各項目に1枚ずつ計上**（合計がデッキ枚数を超え得る旨を注記表示）
5. **タブ状態はhashに持たない**（リロードでデッキタブに戻る。シンプル優先）

## 画面仕様

### タブバー（編集・共有 共通デザイン）

- 編集画面: `.deck-pane` の直前に配置。`🃏 デッキ | 📊 統計`
- 共有画面: `#v-zones` の直前に配置
- 選択中タブに応じて deck-pane（/ v-zones）と stats-pane の hidden を切替

### 統計パネル（stats-pane）の構成（上から。詳細はモック参照）

0. **⚠️警告バナー（スタンダード不適合がある時のみ・第6版）**: `⚠️ スタンダードで使用できない構成です（禁止カード N枚・枚数制限の超過 M種）` + 最下部の適合ブロックへの誘導。パンテオン側のみの不適合では出さない（主眼はスタンダード）
1. **マテリアルデッキ (N枚)**: エレメント別 / タイプ別 / サブタイプ別 / キーワード
2. **メインデッキ (N枚)**: 同構成
3. **サイドボード (N枚)**: 同構成
4. **合計 (N枚)**: 同構成。見出しに内訳注記（`80枚 ＝ マテリアル12＋メイン60＋サイド8（「検討中」は含みません）`）
5. **フォーマット適合（第6版で判定強化）**:
   - **判定対象ゾーン**: スタンダード=メイン+マテリアル+サイド／**パンテオン=メイン+マテリアルのみ**（パンテオンにサイドボードは存在しないため）
   - **判定内容**（フォーマットごとに違反を列挙。違反なしなら `✅ 使用可能`）:
     - 禁止カード（legality limit=0）: `⚠️ 禁止カード 2枚 — Baby Green Slime（緑のベビースライム） ×1・…`
     - メイン同名枚数制限: スタンダード=4枚／パンテオン=1枚（シングルトン）。`⚠️ メイン同名1枚制限の超過 14種 — …`
     - マテリアル同名1枚制限（両フォーマット共通）
   - 長い列挙は**先頭3種+他n種**に短縮（4積みデッキのパンテオン判定はほぼ全カードが該当するため）
   - 注記: `※パンテオンは別途Boon 2枚（Lesser／Greater 各1）が必要です（本ツールでは管理対象外）。パンテオンにサイドボードはありません。`
   - カード名は**「英語（日本語訳）」表記**（cardNameEJ()、未訳は英語のみ）
   - パンテオン専用カード等も legality のlimit=0で bannedFormats() が拾い、不適合要因として列挙される
   - **デッキ最低枚数(60枚)・マテリアル12枚上限は判定しない**（構築途中は常時⚠️でノイズ。ゾーンヘッダのメーターが可視化済み）

各ブロック内のセクション:

- エレメント別・タイプ別: 純CSS横棒グラフ + `WIND（風）` 形式ラベル + 枚数（tabular-nums）
- サブタイプ別（第7版で開閉式に）: `<details class="stat-fold" data-zone="<zone>">` + `<summary>サブタイプ別 N種</summary>` + 枚数降順のテキスト行（`WARRIOR（ウォリアー） 12 ・ SPIRIT（スピリット） 8 ・ …`）。種類が多いのでグラフにしない
  - 初期状態: 描画時に `matchMedia("(max-width: 620px)").matches` でスマホ=閉／PC=開（`open`属性の有無で制御）
  - **再描画時の開閉状態引き継ぎ**: renderStatsInto()でinnerHTML差し替え前に既存の `details.stat-fold` の `data-zone` → open を収集し、差し替え後に再適用（ライブ更新のたびに閉じ直る事故を防ぐ）
  - summaryはh4と同じ見た目+開閉インジケータ（▸/▾はCSSで。ネイティブmarkerは非表示に）
- キーワード: `Floating Memory（フローティングメモリー） N（うち条件付き n枚）` のテキスト行
- 重複計上注記（`※複数のエレメント・タイプ・サブタイプを持つカードは…`）: `.sr-note--multi` 修飾クラスを付け、**≤620pxでは非表示**（第7版。スマホの縦の圧迫回避。Boon注記は素の`.sr-note`のまま表示継続）
- **0枚の分類・0枚のセクションは表示しない**（キーワード0枚なら「キーワード」節ごと非表示）。デッキが空なら「カードがありません」

### 配色（dataviz検証済み。ダーク面 #171a21 に対し明度帯 OKLCH L 0.48–0.67・C≥0.1・コントラスト≥3:1 を全13色でPASS）

- エレメント棒（全13色確定・第3版）:
  `NORM #7b8fd4 / FIRE #cf6d54 / WATER #3f97c4 / WIND #3f9d63 / LUXEM #ad8d2e / CRUX #9b7cd4 / TERA #ad7233 / ARCANE #b667a6 / ASTRA #5b9bd0 / UMBRA #6d64b8 / NEOS #2f9e8f / EXIA #c2678f / EXALTED #8f9b33`
  （未知エレメントはタイプ棒色にフォールバック。13色の相互識別は原理的に不可能なため、識別はラベル直付けが担保）
- タイプ棒: 単色 `#56779e`（同一カテゴリ内の量比較のため色分けしない）
- 各棒はラベル直付けのため色単独に依存しない

### スマホ対応（モックで390px検証済み）

- **≤620px: ラベルを棒の上に折り返す縦積み**（label幅100%・左寄せ、棒+枚数が下段）
- **≤620px: 重複計上注記は非表示**（第7版・`.sr-note--multi { display: none }`）
- **≤620px: サブタイプ別は初期状態で閉じる**（第7版・`<details>`。PCは初期状態で開く）
- それ以上の幅: ラベル右寄せ+棒+枚数の横並び。**ラベル列は最長ラベル（`PHANTASIA（ファンタジア）`等）が折り返さない幅**にする（実装はグリッドの max-content 推奨。モックは15.5em固定で代用）
- 横スクロールが発生しないこと（モックで390px/768px機械検査済み）

## 集計・実装仕様

- **クライアント側集計のみ。API/DB変更なし**（カードは getCard キャッシュに揃っている: tools/deck-builder/app.js の cardCache）
- 集計関数 `computeDeckStats(cards, bySlug)` を app.js に新設。deckData.cards（board=zone, qty持ち）と bySlug(slug→card) から:
  - zone別+合計で elements[] / types[] / subtypes[] を qty 重み付きでカウント
  - **Floating Memory検出（第5版で確定）**: `card.effect_raw`（無ければ `card.effect`）の`*`除去後に**大文字小文字を区別して** `Floating Memory` を検索。
    大文字=キーワード付与（実データ165枚）、小文字のみ=「banish a card with floating memory」等の参照（37枚）のため、大小無視は誤検出になる。
    **条件付き**の内数は `/\[[^\]]+\](\s*\[[^\]]+\])*\s*Floating Memory/`（条件タグ直後の付与。タグ連なりも許容）。
    該当96枚 = [Class Bonus] 88 + [Level N+] 4 + [Vanitas Bonus]/[Sheen 6+]/[Merlin Bonus]/[Ciel Bonus] 各1
  - 禁止判定は `GA_CARD_I18N.bannedFormats(card)`（shared/js/card-i18n.js 実装済み）。
    パンテオン専用カード等も `legality` のlimit=0で表現されているため bannedFormats() で完全カバーされる（exclusiveFormat()の個別使用は不要と実データ108枚で確認済み）
  - **枚数制限（第6版）**: `legality` は禁止（limit:0）のみ持ち枚数上限データは無いため、`FORMAT_RULES` 定数としてルールを実装:
    `STANDARD: { zones: [material,main,side], mainCopyLimit: 4 }` ／ `PANTHEON: { zones: [material,main], mainCopyLimit: 1 }`。
    メインの同名枚数はメインゾーンのみで数える（公式TRG: "Main decks must have ... no more than four copies of any card"。サイドは別途15pt制）。
    マテリアル同名1枚制限は両フォーマット共通。集計はperSlug（slug→ゾーン別qty）を1パスで構築して判定
- **表記**: `GA_CARD_I18N.label("elements"|"types"|"subtypes", value)` が `WIND（風）` 形式を返す（未訳はvalueそのまま）。そのまま使う
- **描画**: 棒グラフは div の width% による純CSS（CSPがself限定のためライブラリ不可。導入も不要）。
  **⚠️CSP注意（第4版）**: `style-src 'self'` は**HTML内のstyle属性も禁止**する。棒の幅・色は `data-w`/`data-c` 属性に載せ、
  innerHTML挿入後に `el.style.width = ...` とCSSOMで適用すること（CSSOM操作はCSPの対象外。既存の.meterバーと同方式）
- **更新フック**: 編集画面は renderZones() の末尾で統計タブ表示中なら再描画（カード追加/削除/枚数変更で自動更新）。共有画面は**統計タブ選択時に遅延描画**（第3版: 閲覧のみで内容が変わらないため、開くたびの描画は不要と判断）
- **共通化**: 集計・描画関数は編集/共有で共用（引数でカード配列を渡す）
- 両面(flip)カードは表面のフィールドで集計（getCardが返す正規形をそのまま使う）
