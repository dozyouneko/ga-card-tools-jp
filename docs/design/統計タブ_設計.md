# タスク#12: デッキ統計タブ 設計書

作成: 2026-07-16（VSCode拡張版セッションで設計。実装はターミナル版で本ファイルを参照）

## 決定済み要件（2026-07-16 ユーザー決定）

- **表示データ**: ①エレメント別枚数 ②タイプ/サブタイプ内訳 ③フォーマット適合チェック
  - コストカーブは今回**見送り**（将来追加は容易。集計関数に1ブロック足すだけ）
- **表示場所**: 編集画面（#view-editor）と共有閲覧画面（#view-deck）の両方
- **UI形式**: タブ切替（「🃏 デッキ｜📊 統計」）。タブ基盤は新規作成

## 設計判断（実装前にユーザー確認済みであること）

1. **集計対象ゾーン**
   - エレメント別・タイプ/サブタイプ別: **メインデッキとマテリアルデッキを別ブロックで集計**。サイドボードは第一弾では対象外、検討中(maybe)は常に除外
   - フォーマット適合: **メイン+マテリアル+サイド**を対象（大会ではサイドも判定対象のため）。検討中は除外
2. **編集画面のタブ挙動**: 検索バー(search-top)は統計タブ表示中も**出したまま**。統計を見ながらカード追加→集計が即時更新される流れを活かす（例:「ノームをあと2枚増やす」調整）
3. **フォーマット表示はスタンダード/パンテオンの2つ**。ドラフトは構築デッキに無意味なので出さない
4. **複数エレメント/サブタイプ持ちのカードは各項目に1枚ずつ計上**（合計がデッキ枚数を超え得る旨を注記）
5. **タブ状態はhashに持たない**（リロードでデッキタブに戻る。シンプル優先）

## 画面仕様

### タブバー（編集・共有 共通デザイン）
- 編集画面: `.deck-pane` の直前に配置。`🃏 デッキ | 📊 統計`
- 共有画面: `#v-zones` の直前に配置
- 選択中タブに応じて deck-pane（/ v-zones）と stats-pane の hidden を切替

### 統計パネル（stats-pane）の構成（上から）
1. **フォーマット適合**: `スタンダード: ✅ 使用可能` / `⚠️ 禁止カード: <和名> ×2, …`
   - パンテオン専用カード等(exclusiveFormat)が入っている場合はスタンダード側の不適合要因として列挙
2. **メインデッキ (N枚)**
   - エレメント別: 純CSS横棒グラフ + `風 12` 形式のラベル（和訳は GA_I18N.label("elements", v)）
   - タイプ別: 同形式（アリー/アクション/フェイズ等）
   - サブタイプ別: 枚数降順のテキスト行（`ノーム 8 ・ 戦士 6 ・ …`）。サブタイプは種類が多いのでグラフにしない
3. **マテリアルデッキ (N枚)**: 同構成（エレメント/タイプ/サブタイプ）
- 0枚の分類は表示しない。デッキが空なら「カードがありません」

## 集計・実装仕様

- **クライアント側集計のみ。API/DB変更なし**（カードは getCard キャッシュに揃っている: tools/deck-builder/app.js の cardCache）
- 集計関数 `computeDeckStats(cards, bySlug)` を app.js に新設。deckData.cards（board=zone, qty持ち）と bySlug(slug→card) から:
  - zone別に elements[] / types[] / subtypes[] を qty 重み付きでカウント
  - 禁止判定は `GA_I18N.bannedFormats(card)` / `GA_I18N.exclusiveFormat(card)`（shared/js/card-i18n.js 実装済み）
- **描画** `renderStats()`: 棒グラフは div の width% による純CSS（CSPがself限定のためライブラリ不可。導入も不要）
- **更新フック**: 編集画面は renderZones() の末尾で統計タブ表示中なら renderStats() を呼ぶ（カード追加/削除/枚数変更で自動更新）。共有画面は openDeckView() の描画後に1回
- **共通化**: 集計・描画関数は編集/共有で共用（引数でカード配列を渡す）
- 両面(flip)カードは表面のフィールドで集計（getCardが返す正規形をそのまま使う）
