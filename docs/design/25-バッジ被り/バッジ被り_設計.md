## 目的

トップのカード検索結果で、左上の `.tl-badge`(「日本語訳あり／未翻訳」)が
**カード画像に印刷されたコスト表示(左上)を恒久的に覆っている**問題を解消する。

背景(issue #25 の実測):

- 翻訳データを持つカードは **2,262件**、検索1ページ目(50件)は **日本語訳あり 50 / 未翻訳 0**。
  「日本語訳あり」は**ほぼ100%常時点灯**しており、カード同士を見分ける情報になっていない
- 一方でコストという**重要情報を恒久的に隠している**。情報価値と遮蔽コストが逆転している
- 逆に「未翻訳」は稀で、**そちらこそ知らせる価値がある**

| 残る「未翻訳」バッジの扱い | **コストを避けて再配置**(画像四隅は全て意味があるため画像外=card-body へ) |
| 同梱範囲 | **format-badge / flip-badge の左下衝突も直す**。案C(`.card-add` の属性隠れ)は**今回対象外**(別issueに残す) |

### ⚠️ format-badge と flip-badge の左下衝突(issueの「未検証」を確定)

両方とも `left:8px; bottom:8px` で**同一座標**。

### 影響範囲は狭い

- バッジ生成は [app.js:173-190](../../../app.js#L173) の `.card-img` innerHTML のみ
- `.tl-badge`/`.tl-yes`/`.tl-no` の出現は **app.js:176 と style.css:256-262 だけ**(grep確認済み)
- **カード個別ページ(`/cards/<slug>/`)・カード詳細モーダル([card-detail.js](../../../shared/js/card-detail.js))には `.tl-badge` は無い**。

- ⇒ **変更対象は [app.js](../../../app.js) と [style.css](../../../style.css) のみ**

## 対処内容

### 変更1: 案A —「日本語訳あり」を出さず、「未翻訳」は画像外へ移す

`.card-img` の overlay から `.tl-badge` を**丸ごと撤去**する。
`translated` が偽のときだけ、`.card-body` のカード名の前に**インラインの「未翻訳」ラベル**を出す。

- **翻訳済み(ほぼ全カード)**: バッジは一切出ない → 左上のコストが見える
- **未翻訳(稀)**: card-body 内(画像の下)に「未翻訳」ラベル → **画像のコストも属性も一切覆わない**

app.js の変更イメージ([app.js:176](../../../app.js#L176) を削除、[app.js:183](../../../app.js#L183) を変更):

```js
// .card-img から 176 行目の <span class="tl-badge"> を削除

// .card-body のカード名の前にラベルを付与
<p class="card-name">${translated ? "" : `<span class="badge-untranslated">未翻訳</span>`}${escapeHtml(jpName(card))}</p>
```

CSS([style.css:256-262](../../../style.css#L256) の `.tl-badge`/`.tl-yes`/`.tl-no` を削除し、下記を追加):

```css
.badge-untranslated {
  display: inline-block; vertical-align: middle;
  margin-right: 6px;
  font-size: .68rem; font-weight: 700;
  color: #fff; background: var(--no);   /* 既存の未翻訳色 #7a6320 を流用 */
  padding: 1px 6px; border-radius: 999px;
}
```

> `.tl-no` は既に `var(--no)` を使っており、色は据え置き(#7a6320)。緑の `.tl-yes`(`--yes`)は
> 「日本語訳あり」ごと消えるため未使用になる。`--yes` 変数自体は `.card-add:hover` 等で使うので残す。

### 変更2: format-badge / flip-badge の左下衝突を解消(縦積み)

左下の2バッジを**フレックスの縦積みコンテナ**でまとめ、0/1/2個いずれでも破綻しないようにする。
片方だけのときは従来どおり `bottom:8px` の位置に1つだけ出て、**浮いた隙間を作らない**。

app.js([app.js:177](../../../app.js#L177) と [app.js:179](../../../app.js#L179) を1つの要素にまとめる):

```js
<div class="badges-bl">
  ${formatBadgeHtml(card)}
  ${back ? `<button class="flip-badge" type="button" title="両面カード：表裏を切り替え" aria-label="裏面を表示">🔄 両面</button>` : ""}
</div>
```

> ⚠️ `.flip-badge` は [app.js:200](../../../app.js#L200) で `cardEl.querySelector(".flip-badge")` されイベントが付く。
> **クラス名・要素種別(button)は変えないこと。** ラップ位置が変わるだけで querySelector は従来どおり効く。
> `formatBadgeHtml(card)` は該当なしのとき空文字を返す前提(既存挙動)。

CSS(`.format-badge`/`.flip-badge` から `position/left/bottom` を外し、コンテナで位置決め):

```css
.badges-bl {
  position: absolute; left: 8px; bottom: 8px;
  display: flex; flex-direction: column-reverse;  /* DOM先頭(format)を下に、flipを上へ積む */
  align-items: flex-start; gap: 6px;
}
.format-badge, .flip-badge { position: static; }  /* ← absolute/left/bottom を撤去 */
```

- `column-reverse` で **format-badge が下**(=従来の bottom:8px 位置)、flip-badge がその上に乗る
- コンテナは中身が空なら潰れる。片方のみなら1つだけが bottom:8px に置かれる
- 右下 `.art-badge`・右上 `.card-add` は無関係で従来位置のまま

### 検証の注意(未翻訳カードの見つけ方)

未翻訳カードは稀で、検索1ページ目には出ない。項目2・4・6 の確認には次のいずれかで**未翻訳/両面/禁止**の
カードを用意する:

- `isTranslated(card)` が偽になるカードを探す(翻訳データ未整備のカード)。見つからなければ
  **一時的に `const translated = false;` に固定**して表示確認 → 確認後に戻す(コミットしない)
- 両面カード・禁止フォーマットカードは既存データに存在するはず(#16フォーマット検索・両面表示の実績あり)。
  実カードで項目4を再現できないときは同様に一時スタブで確認し、根拠をissueに残す

Playwright実駆動は CLAUDE.md「環境の注意」の証明書回避
(`--ignore-certificate-errors` + `ignoreHTTPSErrors: true`)に従う。

## 設計からの逸脱がないことの確認事項(実装担当向け)

- `.tl-badge` を消すことで**カード詳細モーダル/個別ページに副作用が無い**こと(元々存在しないので影響なしのはず)を確認
