## 件数基準について(2026-07-18 API実測)

「スタンダードで使用可」の件数は**全カード数 − スタンダード禁止数**で決まり、新カード追加・禁止改定のたびに変動する:

| 時点 | 全体 | スタンダード禁止 | 使用可(=差) |
|---|---|---|---|
| 2026-07-16(設計書作成時) | 2,237 | 122 | 2,115 |
| 2026-07-18(本チェックリスト作成時) | 2,240 | 122 | **2,118** |

→ **2,118件は正常**(7/16以降に新カード3枚追加。禁止は変化なし)。チェック時は絶対値でなく「全体−禁止=使用可」の整合と Baby Green Slime の除外/包含で判定する。

## チェック項目

### 0. 基本疎通

- [ ] 1. トップページ(カードDB本体)が表示され、カード検索が動く
- [ ] 2. デッキ構築ツールが表示される
- [ ] 3. コンソールにCSP違反・JSエラーが出ていない(F12で確認)

### 1. #16 フォーマット検索

- [ ] 4. カードDB本体の絞り込みに「フォーマット」セレクトがあり、選択肢が6つ(全て/使用可3種/禁止2種)
- [ ] 5. 「スタンダードで使用可」→ 件数が「全体−スタンダード禁止」と一致(2026-07-18時点=2,118件)、Baby Green Slime が**出ない**
- [ ] 6. 「スタンダード禁止」→ 122件、Baby Green Slime が**出る**
- [ ] 7. 日本語検索モードとの整合: 「緑のベビースライム」で和名検索+「スタンダードで使用可」→ 出ない/「スタンダード禁止」→ 出る
- [ ] 8. 他フィルタ(エレメント等)や「もっと見る」(ページング)との併用が正常
- [ ] 9. リセットボタンで「フォーマット(全て)」に戻る
- [ ] 10. デッキ構築ツール側の検索でも同セレクトが動く(4〜6の簡易確認でOK)

### 2. #12 統計タブ(編集画面)

- [ ] 11. デッキ編集画面に「🃏 デッキ|📊 統計」タブが表示され、切替できる
- [ ] 12. 統計タブに マテリアル→メイン→サイド→合計→フォーマット適合 の順で表示される
- [ ] 13. **エレメント別の棒グラフに色が付いている**(本番CSPでdata属性+CSSOM適用が機能している証拠。最重要)
- [ ] 14. カードを追加/削除すると統計タブ表示中でも即時更新される
- [ ] 15. サブタイプ別が初期状態で**閉じている**(PC幅でも)。開いた状態でカード追加→開いたまま更新される
- [ ] 16. 禁止カード(例: Baby Green Slime)入りデッキで ⚠️警告バナー+適合ブロックに「英語(日本語訳)」表記の禁止カード名が出る
- [ ] 17. 同名5枚以上で「メイン同名4枚制限の超過」が出る(スタンダード判定)

### 3. #12 統計タブ(共有画面)

- [ ] 18. 共有リンクを開き、統計タブを選択すると統計が描画される(遅延描画)
- [ ] 19. **読み込み中に統計タブを先押し**しても、読み込み完了後に統計が表示される(空のまま残らない=設計書第4版⑧の確認)

### 5. docs遮断・退行確認
