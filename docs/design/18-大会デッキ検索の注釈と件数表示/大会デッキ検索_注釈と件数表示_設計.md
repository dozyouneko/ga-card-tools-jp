## 目的・要望

一覧画面の説明文が長く、**特にスマホで一覧本体が見えにくい**。件数表示も分かりにくいので整理する。

### ユーザー決定(2026-07-23)

| 論点 | 決定 | 効果(実測) |
|---|---|---|
| 一覧ページの注釈 | **スマホ幅では折りたたむ**(「ℹ️ このページについて」の1行) | 占有 93px → **43px**。表の上端が 355px → **305px** |
| 全件表示時の件数文言 | **「全 420 件」**。絞り込み中のみ「検索結果：8件」 | 検索していないのに「検索結果」と出る違和感を避ける |
| 大会詳細ページの注釈 | **免責文を表の下の注釈へ移す**。上には短い操作ヒントだけ残す | 上部の 94px を大きく削減 |

## 画面モック

[モック_注釈と件数表示.html](モック_注釈と件数表示.html) — ブラウザで直接開ける自己完結HTML。
**スマホ390px幅で現行と変更後を左右に並べて**確認できる(一覧ページ・大会詳細ページの両方)。
0件時の表示も掲載。※見た目の確認用で操作はできない。

## 実測(2026-07-23・Playwright / スマホ390px・PC1280px)

計測対象は `npm run dev`(<http://localhost:3000/tournaments/>)の実画面。

### 1. 件数表示は「無い」のではなく「スマホで見えない」

issueは「**ヒット件数が分からない**」としているが、`#f-hits` は[生成HTMLの594行目](../../../scripts/build-tournament-pages.mjs#L594)に既にあり、
`8 / 420 大会` のように**件数を出している**。問題は**置き場所**だった。

```text
Playwright isVisible("#f-hits")  → false     ← スマホ390px
details#filter-box .open         → false
element.checkVisibility()        → false
```

`#f-hits` は `<details class="filter-box">` の**最後の子**で、
[tournaments.js の `fitViewport()`](../../../tournaments/tournaments.js) がスマホ幅で `box.open = false` にするため、
**枠ごと畳まれて件数が見えなくなる**。PC幅では `open` のままなので見えている(実測: 上端359px)。

さらに悪いことに、畳んだ状態で開催国=日本に絞ると:

| 画面に出ているもの | 値 |
|---|---|
| 絞り込みバッジ(`#f-count`) | `1`(=**適用中の条件数**であって件数ではない) |
| 注釈(`p.sub`) | 「収録 **420** 大会。」 |
| 実際の表示行数 | **8行** |

**画面上の数字(420)と表示内容(8件)が矛盾したまま**になる。これがissueの体感の正体。

### 2. 文言の短縮だけでは効果が薄い(最大25px)

`p.sub` の文言を差し替えて390px幅での高さを実測した。

| 案 | 文字数 | 高さ | 行数 |
|---|---|---|---|
| A 現行 | 81 | 75px | 3行 |
| B 「収録420大会」だけ削る | 71 | **75px** | **3行** |
| C 「主要大会」を省く | 66 | **75px** | **3行** |
| E 中間(SCに略す) | 55 | 50px | 2行 |
| D 大幅短縮「主要大会の提出デッキを日本語訳付きで見られます。」 | 24 | 50px | 2行 |

**66文字までは3行のまま**で、カテゴリ名の列挙を捨てて24文字まで削っても**2行50px止まり**。
短縮で稼げるのは最大25pxしかない。**折りたたみ(25px)のほうが、文言を1文字も削らずに大きく縮む。**

### 3. 折りたたみの効果

`p.sub` を `<details>` に置き換えて実測した。

| 状態 | 注釈の高さ | 表の上端 |
|---|---|---|
| 現行(常時表示) | 75px(margin込み93px) | 355px |
| **折りたたみ・閉** | **25px** | **305px** |
| 折りたたみ・開 | 108px | 388px |

スマホ390px(ビューポート844px)における表より上の内訳:

### 4. 注釈を畳んでもカテゴリ名はページから消えない

issueの「SEO・サイト説明としての役割があるため完全削除は避けたい」を定量確認した。
生成済みの `tournaments/index.html` 内の出現回数:

| キーワード | ページ全体 | うち注釈内 |
|---|---|---|
| Store Championship | 1,217回 | 1回 |
| Regionals | 224回 | 1回 |
| Nationals | 21回 | 1回 |
| Ascent | 10回 | 1回 |

表の「種別」列とカテゴリ絞り込みの選択肢に大量に出現するため、
**注釈を折りたたんでも当該キーワードがページから失われることはない**。
加えて `meta description`([build-tournament-pages.mjs:540](../../../scripts/build-tournament-pages.mjs#L540))に
ほぼ同文が入っている。**今回の方式は本文をDOMから消さない**(閉じた `<details>` に入れるだけ)ので、
そもそも削除にはあたらない。

### 5. 【新規検出】0件に絞ると画面が空になる

## 設計

### A. 一覧ページ: 注釈を折りたたむ

既存の絞り込み枠と**同じ手法**を使う(新しい仕組みを増やさない)。

```html
<details class="about" id="about-box" open>
  <summary>ℹ️ このページについて</summary>
  <p class="sub">主要大会(Store Championship・Regionals・Nationals・Ascent)の提出デッキを日本語訳付きで見られます。</p>
</details>
```

- **本文の文言は変えない。ただし「収録 N 大会。」は削る**(件数表示に一本化する。issue「収録全数の表示は不要」)
- PC幅では `summary { display:none }` + `open` のまま = **現行と同じ1行表示**
  ([CSS 835行目](../../../scripts/build-tournament-pages.mjs#L835) の `.filter-box > summary` と同じ書き方)
- スマホ幅(≤640px)で `summary` を表示し、[tournaments.js の `fitViewport()`](../../../tournaments/tournaments.js) に
  `aboutBox.open = !narrow.matches` を**1行足して**畳む。**既存の `narrow` matchMedia を再利用**する
- JSが動かない場合は開いたまま(現行と同じ)= 劣化しても情報は読める

### B. 一覧ページ: 件数表示を絞り込み枠の外へ出す

`#f-hits` を `</details>` の**直後・表の直前**へ移す。**これが本issueの本質的な修正**。

```html
</details>

<p class="filter-hits" id="f-hits" role="status"></p>

<div class="cp-scroll"><table id="ev-table">
```

- **PC・スマホで同じ置き場所**にする(条件分岐を作らない)
- `role="status"` は現状のまま残す(絞り込みのたびに支援技術へ読み上げられる)
- 表の直上に置くことで「この表が何件か」という対応が明確になる

文言(ユーザー決定):

| 状態 | 表示 |
|---|---|
| 絞り込みなし | `全 420 件` |
| 絞り込みあり | `検索結果：8件` |
| 0件 | `検索結果：0件` |

判定は既存の `apply()` にある `n === rows.length` をそのまま使える
(現行 `hits.textContent = n === rows.length ? \`${rows.length} 大会\` : \`${n} / ${rows.length} 大会\`;`)。

⚠️ **単位は「件」**(issueの指定どおり)。行の実体は大会だが、`全 420 件` / `検索結果：8件` で表記を揃える。

### C. 一覧ページ: 0件のときの文言を追加する

大会詳細ページの `#rank-empty` と**同じ形**で用意する。

```html
<p class="ev-empty" id="ev-empty" hidden>該当する大会がありません。条件を変えてお試しください。</p>
```

- 表の直後に置き、`apply()` で `emptyEl.hidden = n !== 0;` を1行足す
- CSSは既存の `.rank-empty` に相乗りできるならまとめる(実装時に判断してよい)

### D. 大会詳細ページ: 免責文を表の下へ移す

現行の1文を**2つに分けて置き直す**。

| 内容 | 移動先 |
|---|---|
| 「デッキを見る」で提出デッキ(N件)をダイアログ表示します。 | **削る**(ボタン自体が `📄 デッキを見る` と書いてあり自明) |
| カードをクリックすると日本語の詳細が開きます。 | **順位表の上に残す**(短い操作ヒント。デッキダイアログを開く前に要る情報) |
| 当時のフォーマットで提出されたリストのため、現行ルールでの使用可否は判定していません。 | **表の下の注釈へ移す**(「順位は勝ち点順で…」と同じ段落群に合流) |

- 上に残すのは1文(23文字)のみ。**94px → 1行程度**に縮む
- 免責文はページから消さない(**内容は保持し、位置だけ変える**)

## 実装ステップ

対象は [scripts/build-tournament-pages.mjs](../../../scripts/build-tournament-pages.mjs) のみ(HTML・CSS・JSがすべてこの1ファイルから生成される)。

1. **一覧HTML**: `p.sub` を `<details class="about" id="about-box" open>` で包む。「収録 N 大会。」を削る
2. **一覧HTML**: `#f-hits` を `</details>` の直後(表の直前)へ移動する
3. **一覧HTML**: `#ev-empty` を表の直後に追加する
4. **CSS**: `.about > summary { display:none }` と、`@media (max-width:640px)` 内で `display:block`
   (`.filter-box > summary` と同じ形)。`.ev-empty` の体裁
5. **LIST_JS**: `apply()` の件数文言を `全 N 件` / `検索結果：N件` に変更し、`#ev-empty` の出し分けを足す
6. **LIST_JS**: `fitViewport()` に `about-box` の開閉を足す
7. **詳細HTML**: 注釈を分割し、免責文を表の下の注釈群へ移す
8. **再生成**: `node scripts/build-tournament-pages.mjs --no-scan` で**APIを叩かずページのみ再生成**する

⚠️ **ステップ8は `--no-scan` を使うこと。** 素の `npm run build:tournaments` はAPIスキャンが走る。
ページ構成の変更なので既存データからの再生成で足りる(CLAUDE.md「ページ生成ロジックを変えたときはこちら」)。
なお `build()` は `tournaments/` を事前削除しないが、**今回はファイル構成を変えないため `rm -rf` は不要**。
