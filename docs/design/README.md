# docs/design/ の構成

設計成果物は**対処内容(タスク)ごとのサブフォルダ**で管理する。

```text
docs/design/
├── <issue番号>-<タスク名>/   ← タスクごとの設計書・モック
│   ├── ○○_設計.md            (改版履歴表つき。実装済み/実装待ちを明記)
│   └── ○○_モック.html        (必要な場合のみ。自己完結HTML)
└── リリース確認/              ← リリース時の確認資料(日付プレフィックスつき)
    └── YYYY-MM-DD_○○.md
```

## 運用メモ

- 設計はVSCode拡張側セッション、実装はターミナル側セッションが担当(詳細はリポジトリ直下のCLAUDE.md)
- `docs/` 配下は本番配信対象外(`functions/docs/[[path]].js` が全パス404を返す)
- `tmp/` はgitignoreのため設計成果物を置かない
