## 要件（2026-07-16 ユーザー決定）

カード検索の絞り込みに「フォーマット」セレクトを追加する。選択肢は**使用可3種＋禁止2種**：

| 表示 | クエリ |
|---|---|
| フォーマット（全て）※デフォルト | （パラメータなし） |
| スタンダードで使用可 | `legality_format=STANDARD&legality_state=LEGAL` |
| パンテオンで使用可 | `legality_format=PANTHEON&legality_state=LEGAL` |
| ドラフトで使用可 | `legality_format=DRAFT&legality_state=LEGAL` |
| スタンダード禁止 | `legality_format=STANDARD&legality_state=RESTRICTED` |
| パンテオン禁止 | `legality_format=PANTHEON&legality_state=RESTRICTED` |

- 「専用カード」絞り込みは**見送り**（サーバー側で表現不可＝クライアント後処理になりページング件数が不正確化。検索結果の専用バッジ`formatBadgeHtml()`で実用上カバー済み）
- 適用範囲: **カードDB本体とデッキ構築ツールの両方**（共通モジュール`shared/js/card-search.js`への追加で一括対応）
- 集客観点: 禁止改定のたびに「今スタンで使えるカード」「何が禁止されたか」を調べる需要に応える（登録時2026-07-16の判断を踏襲）

## API仕様（2026-07-16 実測確認済み）

- `GET api.gatcg.com/cards/search` は `legality_format`（STANDARD/PANTHEON/DRAFT）× `legality_state`（ANY=デフォルト/LEGAL/RESTRICTED）を受け付け、**サーバー側で絞り込む**（ページング・total_cardsが正確なまま）
- 実測: STANDARD×LEGAL全体=2,115件（全2,237−スタンダード禁止122とスナップショット完全一致）／STANDARD×RESTRICTED×name=slime → Baby Green Slime等の禁止2枚のみ
- 他パラメータ（name/element/prefix/sort等）と併用可
- ※`legality_limit`（枚数指定）もあるが今回は使わない。パラメータ名は`legality_state`（`legality_status`ではない）
- ⚠️APIの癖: `page_size=1`だと`total_cards`が正しく返らない（現実装はpage_size=50なので影響なし。検証時の注意）
- 出典: api-docs.gatcg.com/endpoints/multiple-cards

## UI仕様

- 各ページの絞り込み行に `<select>` を1つ追加（既存セレクトと同じ見た目・同じ折りたたみ内）
  - カードDB本体: `index.html` の `#filter-row` に `id="f-format"`（f-subtypeとf-setの間を想定）
  - デッキ構築ツール: `tools/deck-builder/index.html` の `#s-filter-row` に `id="s-format"`（s-subtypeとs-setの間を想定）
- デフォルトoption: `フォーマット（全て）`。選択肢の文言は上表のとおり（日本語のみ。データ列挙値ではなく意味的フィルタのため「KEY（訳）」形式にしない）
- モック不要（既存セレクト1個の追加のみ）

## 実装仕様

### shared/js/card-search.js

1. **選択肢定数とfill関数を追加**（両ページで同一選択肢を保証）:
   ```js
   const FORMAT_FILTERS = [
     ["STANDARD:LEGAL", "スタンダードで使用可"],
     ["PANTHEON:LEGAL", "パンテオンで使用可"],
     ["DRAFT:LEGAL", "ドラフトで使用可"],
     ["STANDARD:RESTRICTED", "スタンダード禁止"],
     ["PANTHEON:RESTRICTED", "パンテオン禁止"],
   ];
   function fillFormatSelect(select) { /* FORMAT_FILTERSからoption生成 */ }
   ```
   selectのvalueは `"<FORMAT>:<STATE>"` 形式。exportに`fillFormatSelect`を追加
2. **buildQuery()**: `els.format` の値があれば `":"` で分割し `legality_format` と `legality_state` をセット
3. **matchesActiveFilters()**（日本語検索モードのクライアント側フィルタ）に同じ意味論を追加:
   ```js
   if (val(els.format)) {
     const [fmt, state] = val(els.format).split(":");
     const banned = GA_CARD_I18N.bannedFormats(card).includes(fmt);
     if (state === "LEGAL" ? banned : !banned) return false;
   }
   ```
   （`bannedFormats()`はlimit=0判定＝APIのRESTRICTEDと同義。専用カードも両側で同じ扱いになることを#12第6版で実データ108枚確認済み）

### 各ページ

- `index.html`／`tools/deck-builder/index.html`: セレクト要素を追加
- `app.js`／`tools/deck-builder/app.js`: `el.`マップに追加 → `GA_CARD_SEARCH.create({ els: { …, format: … } })` に配線 → 初期化時に `fillFormatSelect()` 呼び出し
- **リセット処理への追加を忘れない**（両ページの検索リセットがこのセレクトも「全て」に戻すこと）

## 検証項目（実装時。Playwright実駆動）

1. 「スタンダードで使用可」→ Baby Green Slime（スタンダード禁止・和訳あり）が結果に出ない。全体件数が2,115前後（禁止改定で変動しうる）
2. 「スタンダード禁止」→ 禁止カードのみ約122件。Baby Green Slimeが出る
3. エレメント等の他フィルタ・並び替え・「もっと見る」（ページング）との併用が正常
4. **日本語検索モードとの整合**: 禁止カードの和名（例: 緑のベビースライム）で検索+「スタンダードで使用可」→ 出ない／「スタンダード禁止」→ 出る
5. カードDB本体とデッキ構築ツールの**両方**で動作。リセットボタンで「全て」に戻る
6. フォーマット未選択時は従来と完全に同一の挙動（後方互換）
