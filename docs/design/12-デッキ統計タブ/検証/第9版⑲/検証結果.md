# 第9版⑲ 検証結果 — 統計パネルの `max-width: 880px`

- **検証日**: 2026-07-18
- **対象変更**: `tools/deck-builder/style.css` の `.stats-pane` に `max-width: 880px;`(左寄せ=`margin auto`なし)を追加
- **目的**: ワイドPC画面(1400px超)で `.sr-track{flex:1}` が画面幅まで伸び、棒・テキスト行・フォーマット適合ブロックが長すぎた問題(設計書 第9版⑲)の解消を確認する
- **判定**: ✅ **ALL PASS**(全8項目)

## 検証方法

Playwright(Chromium headless)で本番ページを実際に読み込み、`renderStatsInto` が生成するのと**同一クラスのマークアップ**を `#ed-pane-stats` に注入して、各画面幅で `getBoundingClientRect()` により実測した。

- ページ: `tools/deck-builder/index.html`(本番 `style.css` をそのまま適用)
- 注入DOM: エレメント別/タイプ別の横棒、サブタイプ別`<details>`(初期閉)、キーワード、フォーマット適合(長い違反列挙)を含む代表的な統計内容
- 棒の幅・色は本番同様 CSSOM(`el.style.width`)で適用(CSP `style-src 'self'` 対応の実装を再現)
- スクリプト: [`verify.mjs`](./verify.mjs)(再実行手順はファイル冒頭のコメント参照)

> 注: `app.js` はバックエンドAPI(`/api/decks` 等)を叩くためローカルでは失敗するが、統計DOMを直接注入して測定するため検証結果には影響しない。実測は描画結果(レイアウト)に対して行っている。

- 最長ラベル `PHANTASIA（ファンタジア）` は全幅で折り返しなし
