## 目的

7/16以降に公式APIへ追加されたP26(2026プロモ)新カード3枚を和訳し、「全カード翻訳済み」状態を回復する。あわせてカード個別ページ・セットページ・sitemapへ反映する。

## 調査結果(2026-07-19)

### データ構造と読み込み

- 翻訳は `window.GA_I18N.cards` に slug キーで登録。セット別ファイルは `data/tl/*.js`(現37ファイル)
- 読み込み箇所は **`index.html` と `tools/deck-builder/index.html` の2ファイル**(`translations.js` の後に `<script>` 列挙)
- `scripts/build-card-pages.mjs` は **index.htmlのscriptタグ列をそのまま読んで同順ロード**するため、index.htmlに追加すれば静的ページ生成にも自動反映される

### 既存P26カードとの関係

- `sets/p26/` の既存7枚(trump-set等)は**再録元セットのtlファイルで既訳**。受け皿がないのは新規3枚のみ
- → **`data/tl/p26.js` を新規作成し、この3枚だけを収録**する(p23/p24/p25と同じ命名・同じファイル構造。ヘッダコメント「// カード名 [TYPE] ELEMENT」形式もp25.jsに倣う)

### ページ生成・sitemap

- `node scripts/build-card-pages.mjs --refresh` **一発で全て更新される**: APIスナップショット更新 → `cards/<slug>/`(新規3ページ) → `sets/p26/`(3枚追加) → 索引 → `sitemap.xml`
- 検証補助: `node scripts/scaffold.mjs P26` で未訳カード一覧が出る(実装後に0枚になることを確認)

## 訳文案

先例: ディヴァインレリック(imperial-seal等)・アセンダント(神化の儀式)・トリスタン・リネージュ(tristan-shadowdancer)・不吉な影トークン(既存Tristan系)。表記はTRANSLATION.md準拠。

### 1. Dinah, Lost Spirit(CHAMPION / SPIRIT / NORM / L0 / LIFE 15)

> EN: On Enter: Draw seven cards. / flavor: "A little whimsy never hurts!"

```js
"dinah-lost-spirit": {
  name: "ダイナ、迷い込んだ精霊",
  effect: "**登場時：** カードを7枚引く。",
  flavor: "「ちょっとした気まぐれも、悪くないでしょ！」",
},
```

※ Dinah は『不思議の国のアリス』のアリスの飼い猫由来(P26はアリス系プロモ)。「ダイナ」表記とする。

### 2. Transcendental Rite(REGALIA/ITEM / MAGE / ARTIFACT / NORM)

> EN: Divine Relic / Banish Transcendental Rite: All basic elements are enabled for you until end of turn.

効果は既訳の「皇帝の玉璽」(imperial-seal)と完全同一 → **同文を流用**:

```js
"transcendental-rite": {
  name: "超越の儀式",
  effect:
    "ディヴァインレリック *（このキーワードを持つカードはマテリアルデッキに1枚しか入れられない。）*\n\n" +
    "「超越の儀式」を追放する：ターン終了時まで、すべての基本エレメントがあなたにとって有効になる。*（火・水・風が基本エレメントである。）*",
  flavor: "「これにより、汝の力は疑いなきものとなる。」",
},
```

※ 既存の「神化の儀式(Rite of...)」「秘蹟の儀式」と同系の Rite=「儀式」で統一。

### 3. Tristan, Ascendant Shadow(CHAMPION / ASSASSIN / HUMAN / NORM / L4 / LIFE 28)

> EN: Tristan Lineage / (2025): Summon any amount of Ominous Shadow tokens. For the rest of the game, attack cards you own have "On Hit: You may rest and declare an attack with a phantasial ally you control. If you do, put this card in that attacker's intent." Activate this ability only if Tristan is an Ascendant.
> flavor: Taylor "boeboe" Boe, Worlds 2025 Champion

```js
"tristan-ascendant-shadow": {
  name: "トリスタン、昇華せし影",
  effect:
    "**トリスタン・リネージュ** *（「トリスタン、昇華せし影」は、前のレベルの「トリスタン」チャンピオンからレベルアップさせなければならない。）*\n\n" +
    "(2025)：不吉な影トークンを望む数**召喚**する。ゲームの残りの間、あなたがオーナーであるアタックカードは「**ヒット時：** あなたのコントロールするファンタジアのアライを1体レストして、そのアライでアタックを宣言してよい。そうしたなら、このカードをそのアタッカーのインテントに置く。」を持つ。この能力は、トリスタンがアセンダントである場合にのみ起動できる。",
  // flavor キーは置かない(英語原文フォールバック。世界王者クレジットは訳さない方針)
},
```

(第3版訂正: 初版訳文案の「命中時」→**ヒット時**(TRANSLATION.md対訳表・使用83件)、「ファンタジアル」→**ファンタジア**(既存49件)、(2025)/アセンダントの太字解除(lorraine-ascendant-wingsの書式)。いずれも実装側の照合が正しかった。カード名は第3版ユーザー決定②で**「昇華せし影」**=姉妹カード「ロレイン、昇華せし翼」の命名規則に統一)

- ✅ **①確定(2026-07-19ユーザー決定)**: 世界王者クレジットのフレーバーは**原文のまま**(クレジットは固有名詞扱いとして訳さない)。今後同種のクレジットフレーバーが出た場合もこの方針に従う
- (2025)はアセンダント関連の起動コスト表記(数値2025)。原文表記のまま `**(2025)：**` とする
- 用語(命中時/レスト/インテント/ファンタジアル等)は既訳のトリスタン系カードの表記に実装時に照合すること

## 実装ステップ(初回実装時のもの・7e00f02で完了)

1. `data/tl/p26.js` を新規作成(上記3枚。p25.jsの構造・ヘッダコメント形式に倣う)
2. `index.html`・`tools/deck-builder/index.html` の2箇所に `<script src="data/tl/p26.js"></script>` を追加(既存tl群の末尾)
3. `node scripts/build-card-pages.mjs --refresh` を実行(cards/3ページ新規・sets/p26更新・索引・sitemap.xml再生成)
4. 検証(下記)がすべてPASSしたらコミット。コミットメッセージに `closes #1`

## 検証項目(実装時)

1. `node scripts/scaffold.mjs P26` → 未訳0枚
2. `npm run validate` PASS・`node scripts/check-terminology.mjs P26` で用語ゆれなし
3. カードDB(ローカル)で3枚が日本語訳ありで表示される(「日本語訳あり」バッジ・詳細モーダルの訳文)。デッキ構築ツールの検索でも同様
4. `cards/dinah-lost-spirit/` 等3ページが生成され表示できる。`sets/p26/` に3枚が追加されている(計10枚)
5. `sitemap.xml` に3枚のURLが追加されている(全体URL数が+3)
6. 既存ページの差分が意図外に出ていないこと(`git diff --stat` で確認。--refreshによる他カードの原文更新が混ざった場合は内容を確認して判断)
