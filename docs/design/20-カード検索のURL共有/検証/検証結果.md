## 補足（設計担当への情報）

- **検証項目14の判定方法を設計から変えた**。設計は「直前に自分が書いたクエリと同じなら何もしない」だったが、
  変数で最後の書き込みを覚える方式にすると、外部要因（他コードの `pushState` 等）で
  変数とURLがずれたときに追従できない。**`queryString()`（現在の画面から組み立てたクエリ）と
  `location.search` を比較する**方式にした。意味は「画面が既にURLどおりなら何もしない」で、
  設計意図（空振りでAPIを叩かない）はそのまま満たす。保持する状態が減り、
  `saveQuery()` と同じ組み立て関数を使うため両者がずれない
- **スクリーンショット**: [共有URL_1280px.png](共有URL_1280px.png) / [共有URL_390px.png](共有URL_390px.png)
  （いずれも `?element=NORM&element=WIND&subtype=CLERIC&subtype=SPELL&subtype_op=AND`）
  - PC(1280px): 復元でエレメント・サブタイプの2グループが開き、**カード一覧が約730px下**に押し出される（Z2の判断材料）
  - スマホ(390px): 絞り込みパネル自体が畳まれているため、**開いたグループは見えずバッジ「4」だけが見える**。
    タップして開くと復元済みの状態が見える（Z3の判断材料）
- **デッキ構築ツールへの影響なし**を確認。`tools/deck-builder/app.js` は
  `fillSelect` / `fillFormatSelect` / `fillSetSelect` / `create` しか使っておらず、
  今回追加した `setValues()` / `setMode()` / `setKeyOf()` / `setIndexOf()` を呼ぶ箇所はない（追加のみで既存挙動は不変）
